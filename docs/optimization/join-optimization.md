# Join 优化

## 1. 它是什么？

Join 优化是降低表关联时数据移动、数据膨胀和内存压力的方法。

## 2. 它解决什么问题？

它解决 Join 慢、Join 后数据量暴增、Shuffle 大和倾斜问题。

## 3. 它在整个流程中的位置？

Join 广泛出现在 DWD 补维、DWS 汇总和 ADS 宽表构建。

## 4. 底层原理是什么？

Join 性能取决于关联键分布、表大小、Join 类型、过滤条件和物理执行策略。

## 5. 典型使用场景

事实表关联维表、大表关联大表、明细补充属性。

## 6. 常见问题

关联条件不唯一、维表多版本未过滤、大表 Shuffle 倾斜。

## 7. 优化方案

广播小表、过滤后 Join、控制维表唯一性、处理热点 Key。

## 8. 和其他技术的区别

Join 优化是 SQL 优化的重要子集，更关注表间关系。
