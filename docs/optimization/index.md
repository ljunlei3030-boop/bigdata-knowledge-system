# 性能优化概览

## 1. 它是什么？

性能优化是对 SQL、Join、Shuffle、内存、并行度和数据布局的系统调优。

## 2. 它解决什么问题？

它解决任务慢、查询慢、资源浪费、GC 高和 OOM 问题。

## 3. 它在整个流程中的位置？

它贯穿 Spark 加工、Flink 链路、湖表读写和 OLAP 查询。

## 4. 底层原理是什么？

先定位瓶颈，再从数据量、执行计划、分布、资源和存储布局优化。

## 5. 典型使用场景

Spark SQL 慢、Join 数据膨胀、Shuffle 过大、查询扫描过多。

## 6. 常见问题

不看执行计划直接改参数，容易掩盖根因。

## 7. 优化方案

先查执行计划和监控指标，再选择 SQL 改写、广播、分区、缓存或资源调整。

## 8. 和其他技术的区别

性能优化是跨技术的方法，不局限于 Spark。

## 总结输出

性能优化要从现象进入证据，再从证据回到数据、计划和资源。
