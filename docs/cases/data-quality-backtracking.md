# 案例：数据质量异常回溯

## 1. 现象

某核心指标日环比异常下降，业务侧反馈报表数据明显偏低。

## 2. 初步判断

可能是采集缺失、入湖延迟、加工任务失败、过滤条件变化或上游口径变更。

## 3. 排查过程

先查看 ADS 表质量规则，确认异常分区。沿血缘向上追踪 DWS、DWD 和 ODS，发现 ODS 某来源日志量下降。继续查看采集链路，发现 Kafka 某 Topic 消费延迟异常。

## 4. 根因分析

采集端服务变更导致部分事件未按原 Schema 上报，下游清洗逻辑过滤了异常记录。

## 5. 解决方案

修复采集 Schema，回放 Kafka 数据，重跑 ODS 到 ADS 链路，并补充 Schema 变更校验。

## 6. 结果验证

补数后指标恢复正常，质量规则通过，业务报表重新发布。

## 7. 经验沉淀

质量异常要沿血缘逐层回溯，不能只在 ADS 层修补结果。

## 总结输出

数据质量回溯依赖元数据、血缘、质量规则和调度记录，治理能力越完整，定位越快。
