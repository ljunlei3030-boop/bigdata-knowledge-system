# 案例：Join 后数据量暴增排查

## 1. 现象

一张 DWD 明细表关联维表后，输出数据量比输入数据量放大数倍。

## 2. 初步判断

怀疑 Join Key 不唯一、维表存在多版本记录或关联条件缺失。

## 3. 排查过程

统计维表 Join Key 的重复数量，发现同一个业务主键存在多条有效记录。继续检查维表生命周期字段，发现没有过滤最新版本。

## 4. 根因分析

事实表与多版本维表直接 Join，造成一对多关联，导致数据量膨胀。

## 5. 解决方案

先在维表中按业务主键取最新有效记录，再与事实表关联，并增加唯一性质量校验。

## 6. 结果验证

输出数据量与事实表粒度一致，重复指标恢复正常。

## 7. 经验沉淀

Join 前必须确认关联键粒度和唯一性，不能只检查 SQL 是否能跑通。

## 总结输出

Join 数据膨胀的核心通常是业务粒度不一致，排查要先验证关联键唯一性。
