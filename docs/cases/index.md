# 实战案例概览

实战案例用于沉淀真实项目中的排查过程。每个案例都应从现象出发，经过初步判断、排查过程、根因分析、解决方案、结果验证和经验沉淀，最后形成可复用的总结输出。

## 案例列表

- [Spark SQL 运行缓慢排查](/cases/spark-slow-sql)
- [Executor 频繁 GC 排查](/cases/spark-gc-high)
- [Join 后数据量暴增排查](/cases/join-data-explosion)
- [数据质量异常回溯](/cases/data-quality-backtracking)

## 总结输出

案例的价值不是记录某次故障，而是把排查路径和判断依据沉淀下来，让类似问题下次可以更快定位。
