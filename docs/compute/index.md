# 数据计算概览

## 1. 它是什么？

数据计算负责对采集和存储的数据进行清洗、聚合、关联、建模和查询。

## 2. 它解决什么问题？

它把原始数据转化为可分析、可服务、可复用的数据资产。

## 3. 它在整个流程中的位置？

计算位于存储和服务之间，贯穿离线加工、实时加工和 OLAP 查询。

## 4. 底层原理是什么？

Spark 负责批处理和复杂 SQL，Flink 负责实时流处理，ClickHouse / ByteHouse 负责高性能查询。

## 5. 典型使用场景

数据清洗、宽表构建、指标汇总、实时计算、交互式分析。

## 6. 常见问题

SQL 慢、Shuffle 大、数据倾斜、内存不足、查询延迟高。

## 7. 优化方案

分析执行计划、减少扫描、优化 Join、控制 Shuffle、选择合适引擎。

## 8. 和其他技术的区别

不同计算引擎适合不同延迟、吞吐和查询模式。
