# Kafka

## 1. 它是什么？

Kafka 是分布式消息队列，用于承接高吞吐事件流和日志流。

## 2. 它解决什么问题？

它解决上下游解耦、削峰、缓冲和数据回放问题。

## 3. 它在整个流程中的位置？

Kafka 通常位于采集层和实时计算层之间，上游是日志或业务事件，下游是 Flink、Spark Streaming 或入湖任务。

## 4. 底层原理是什么？

Topic 被拆分为多个 Partition，生产者写入消息，消费者按消费组读取并维护 offset。

## 5. 典型使用场景

日志采集、实时指标、事件驱动、湖仓入湖缓冲。

## 6. 常见问题

消息堆积、重复消费、分区不均、延迟升高和 offset 管理不当。

## 7. 优化方案

合理规划 Topic、Partition、消费并行度、保留策略和监控告警。

## 8. 和其他技术的区别

Kafka 负责数据流转，不负责长期数据建模和复杂分析。

Kafka 是实时链路的缓冲层，核心价值是高吞吐、可回放和上下游解耦。
