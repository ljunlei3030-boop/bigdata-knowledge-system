# 埋点

## 1. 它是什么？

埋点是在业务系统中记录用户行为、页面访问、点击和业务事件的数据采集方式。

## 2. 它解决什么问题？

它解决行为数据从业务前端或服务端进入分析链路的问题。

## 3. 它在整个流程中的位置？

埋点数据通常进入 Kafka，再落入 ODS 明细层或 Paimon 湖表。

## 4. 底层原理是什么？

客户端或服务端按照事件模型生成日志，采集服务接收后写入消息队列，后续由 Flink 或 Spark 消费。

## 5. 典型使用场景

用户行为分析、转化漏斗、推荐特征、实时监控。

## 6. 常见问题

字段缺失、事件重复、客户端时间不准、版本变更未同步。

## 7. 优化方案

制定事件规范、统一字段字典、增加去重键和采集质量监控。

## 8. 和其他技术的区别

埋点偏行为事件采集，Flink CDC 偏数据库变更采集。

## 总结输出

埋点要同时关注业务语义和工程可靠性，核心是事件定义清晰、数据可追踪、质量可监控。
