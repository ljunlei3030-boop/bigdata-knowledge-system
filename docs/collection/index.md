# 数据采集概览

## 1. 它是什么？

数据采集是数据平台的入口，负责把业务日志、行为事件和数据库变更接入后续存储与计算链路。

## 2. 它解决什么问题？

它解决数据从业务系统进入数据平台的问题，并保证数据可回放、可追踪、可校验。

## 3. 它在整个流程中的位置？

数据采集位于 ODS 或入湖之前，直接影响后续建模、治理和指标质量。

## 4. 底层原理是什么？

常见链路包括埋点上报、日志采集、Kafka 缓冲、Flink CDC 捕获数据库变更。

## 5. 典型使用场景

- 业务行为日志进入 Kafka。
- 数据库 binlog 通过 Flink CDC 入湖。
- 客户端埋点进入明细表。

## 6. 常见问题

数据丢失、重复、延迟、字段变更和采集口径不一致。

## 7. 优化方案

建立采集规范、Schema 管理、延迟监控、补偿机制和质量校验。

## 8. 和其他技术的区别

采集关注数据进入平台，计算关注数据加工，治理关注数据可信。

## 9. 关联知识

- [Flink CDC](/collection/flink-cdc)
- [Kafka](/collection/kafka)

## 总结输出

数据采集质量决定后续数据链路上限，必须从源头保证完整、准确和可追踪。
