# ODS / DWD / DWS / ADS

## 1. 它是什么？

ODS、DWD、DWS、ADS 是数据仓库常见分层方式。

## 2. 它解决什么问题？

它通过分层让原始数据、明细数据、汇总数据和服务数据职责清晰。

## 3. 它在整个流程中的位置？

分层模型贯穿数据从入湖到服务的全过程。

## 4. 底层原理是什么？

ODS 保留原始数据，DWD 清洗明细，DWS 汇总公共主题，ADS 面向应用服务。

## 5. 典型使用场景

订单明细清洗、用户主题汇总、经营看板指标输出。

## 6. 常见问题

跨层引用混乱、ADS 逻辑回流、DWD 不稳定、DWS 不复用。

## 7. 优化方案

明确每层职责，控制依赖方向，统一指标口径。

## 8. 和其他技术的区别

分层是建模方法，可以落在 Hive、Paimon、ClickHouse 等不同技术上。

## 总结输出

分层模型的核心是让数据从原始到服务逐步沉淀，减少重复加工和口径分裂。
