# 数据建模概览

## 1. 它是什么？

数据建模是把原始数据组织成稳定、清晰、可复用的数据资产的过程。

## 2. 它解决什么问题？

它解决口径不一致、重复加工、表难复用和指标难追踪的问题。

## 3. 它在整个流程中的位置？

建模位于采集入湖之后、数据服务之前。

## 4. 底层原理是什么？

通过数据域、分层模型、维度建模和指标体系组织数据。

## 5. 典型使用场景

ODS、DWD、DWS、ADS 分层建设，主题域模型建设，宽表设计。

## 6. 常见问题

层次混乱、维度重复、指标口径分散、宽表过宽。

## 7. 优化方案

统一分层规范、沉淀公共维度、管理指标口径和生命周期。

## 8. 和其他技术的区别

建模关注业务语义和数据组织，不是单纯 SQL 开发。

## 总结输出

数据建模让数据从可用变成好用，核心是分层、复用、口径和治理。
