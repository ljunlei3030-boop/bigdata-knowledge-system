# 数据域划分

## 1. 它是什么？

数据域是按业务主题对数据资产进行归类的方法。

## 2. 它解决什么问题？

它解决表和指标缺少业务边界、难以管理和复用的问题。

## 3. 它在整个流程中的位置？

数据域通常在建模设计初期确定，影响后续分层和指标体系。

## 4. 底层原理是什么？

按照业务过程、组织边界和分析主题划分订单、用户、商品、交易、履约等域。

## 5. 典型使用场景

主题模型建设、指标归属、数据资产目录。

## 6. 常见问题

域划分过细、交叉归属、负责人不明确。

## 7. 优化方案

以核心业务过程为主线，并给每个域明确负责人和边界。

## 8. 和其他技术的区别

数据域是业务组织方式，不是存储目录。

## 总结输出

数据域划分让数据资产有业务归属，是指标治理和模型复用的基础。
