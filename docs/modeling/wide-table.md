# 宽表设计

## 1. 它是什么？

宽表是把多个维度和指标预先关联到一张表中，降低查询复杂度的数据模型。

## 2. 它解决什么问题？

它解决查询侧频繁 Join、口径重复实现和响应慢的问题。

## 3. 它在整个流程中的位置？

宽表常位于 DWS 或 ADS 层，面向分析和服务。

## 4. 底层原理是什么？

通过提前加工和冗余字段，把查询时计算转移到离线或准实时加工阶段。

## 5. 典型使用场景

BI 看板、用户画像、订单明细分析、指标服务。

## 6. 常见问题

表过宽、更新成本高、口径固化、存储膨胀。

## 7. 优化方案

控制字段范围，区分通用宽表和应用宽表，管理生命周期。

## 8. 和其他技术的区别

宽表是建模形态，可落在 Hive、Paimon、ClickHouse 等技术上。

## 总结输出

宽表用冗余换查询效率，适合稳定高频场景，但需要控制范围和维护成本。
