# 元数据管理

## 1. 它是什么？

元数据管理是对表、字段、分区、血缘、负责人、口径和生命周期等信息的管理。

## 2. 它解决什么问题？

它解决数据是什么、谁负责、怎么用和是否可信的问题。

## 3. 它在整个流程中的位置？

它贯穿采集、入湖、建模、服务和治理。

## 4. 底层原理是什么？

通过采集 metastore、调度任务、SQL 血缘和人工补充信息形成数据资产目录。

## 5. 典型使用场景

查找表含义、确认字段口径、定位负责人、评估表生命周期。

## 6. 常见问题

字段说明缺失、负责人不清、元数据更新滞后。

## 7. 优化方案

自动采集技术元数据，人工维护业务元数据，定期校验有效性。

## 8. 和其他技术的区别

元数据是治理基础，血缘、质量和权限都依赖它。
