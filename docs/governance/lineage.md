# 血缘分析

## 1. 它是什么？

血缘分析描述数据从来源到产出的依赖关系。

## 2. 它解决什么问题？

它解决指标异常时如何追踪上游、变更时如何评估影响的问题。

## 3. 它在整个流程中的位置？

血缘贯穿采集、入湖、建模、加工和服务。

## 4. 底层原理是什么？

通过解析 SQL、调度依赖和表读写关系构建表级、字段级血缘。

## 5. 典型使用场景

质量异常回溯、任务变更影响分析、核心指标链路梳理。

## 6. 常见问题

动态 SQL 难解析、字段级血缘缺失、人工链路未登记。

## 7. 优化方案

结合 SQL 解析、任务调度和人工补充，优先保障核心链路。

## 8. 和其他技术的区别

血缘关注数据流向，元数据关注数据定义。

## 9. 关联知识

- [数据质量](/governance/quality)
- [数据质量异常回溯](/cases/data-quality-backtracking)

## 总结输出

血缘让数据问题可追踪，让变更影响可评估，是治理闭环的关键。
