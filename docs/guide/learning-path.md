# 学习路径

## 1. 它是什么？

学习路径用于把大数据技术点串成可连续阅读的路线。

## 2. 它解决什么问题？

它帮助从单点问题进入系统知识，避免只会处理局部现象。

## 3. 它在整个流程中的位置？

学习路径连接首页、技术专题、问题库和实战案例。

## 4. 底层原理是什么？

按照数据平台链路组织：采集进入湖仓，计算完成加工，建模沉淀资产，治理保障可信，优化提升稳定性。

## 5. 典型使用场景

### 路径一：Spark 执行原理

Spark SQL → Catalyst → DAG → Stage → Task → Executor → Shuffle → 结果输出

### 路径二：湖仓一体架构

Kafka → Flink CDC → Paimon → Spark → ByteHouse → DolphinScheduler → 数据治理

### 路径三：数据建模与治理

数据域 → ODS → DWD → DWS → ADS → 指标体系 → 元数据 → 血缘 → 质量校验

### 路径四：性能问题排查

SQL 慢 → 执行计划 → Spark UI → Shuffle → 数据倾斜 → GC / OOM → 参数优化

## 6. 常见问题

学习路径不是固定顺序，可以根据当前问题从对应专题切入。

## 7. 优化方案

每次新增文章时都应补充关联知识，让路径越来越完整。

## 8. 和其他技术的区别

它不是技术实现，而是知识库的阅读和扩展方式。
