# 问题整理模板

## 1. 它是什么？

问题整理模板用于统一问题库文章结构。

## 2. 它解决什么问题？

它避免问题只记录临时答案，让每次排查都沉淀为可复用知识。

## 3. 它在整个流程中的位置？

它用于 Spark、Flink、Paimon、ClickHouse、建模、治理和性能优化问题的复盘。

## 4. 底层原理是什么？

每个问题都按答案、原理、场景、排查、方案、代价、关联知识和总结输出组织。

## 5. 典型使用场景

```md
# 问题：这里写具体问题

## 答案
## 背后原理
## 发生场景
## 排查方式
## 解决方案
## 方案代价
## 关联知识
## 总结输出
```

## 6. 常见问题

不要只写“怎么修”，还要说明为什么这样修以及可能带来的副作用。

## 7. 优化方案

关键问题应补充 Spark UI 截图、执行计划、SQL 示例或链路图。

## 8. 和其他技术的区别

模板服务于知识沉淀，不替代具体技术分析。

## 9. 关联知识

- [问题库](/questions/)

## 总结输出

一个好的问题页面应能让读者理解问题本身、产生原因、可选方案和方案代价。
