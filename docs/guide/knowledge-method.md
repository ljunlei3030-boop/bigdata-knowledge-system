# 知识体系方法

## 1. 它是什么？

知识体系方法是把技术点从“单个答案”整理为“模块、链路、问题、案例”的方法。

## 2. 它解决什么问题？

它避免知识停留在碎片化记录中，让问题可以被复用、关联和持续演进。

## 3. 它在整个流程中的位置？

它贯穿文章新增、专题扩展、问题复盘和案例沉淀全过程。

## 4. 底层原理是什么？

先确定技术点所在的数据链路，再解释它解决的问题、底层机制、典型场景、常见风险和可选方案。

## 5. 典型使用场景

- 整理一次 Spark SQL 慢查询。
- 复盘一次数据质量异常。
- 梳理 Paimon 在湖仓链路中的作用。

## 6. 常见问题

只写命令和结论会导致后续难以迁移。需要说明背景、约束和代价。

## 7. 优化方案

用 Mermaid 画链路，用表格比较方案，用关联链接连接前后知识点。

## 8. 和其他技术的区别

它关注知识组织方式，不限定具体技术栈。

## 从一个问题沉淀为知识体系

查一个技术问题时，不要只记录答案，而是把它放回技术体系中。

例如：

问题：Spark 为什么会发生数据倾斜？

它在体系中的位置是：

```text
数据计算
└── Spark
    └── Shuffle
        └── 数据倾斜
            ├── group by 倾斜
            ├── join 倾斜
            ├── distinct 倾斜
            └── repartition 倾斜
```

对应的学习链路是：

```text
SQL 慢
↓
看 Spark UI
↓
定位 Stage
↓
查看 Task 分布
↓
查看 Shuffle Read
↓
判断是否数据倾斜
↓
选择优化方案
```

这样一个问题就不再是孤立答案，而是能挂到 Spark 执行流程、Shuffle 原理、性能优化和排查方法下面。

## 总结输出

系统化理解来自稳定的整理结构：位置、原理、场景、方案、代价和关联知识缺一不可。
