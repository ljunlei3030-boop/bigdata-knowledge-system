# 元数据映射

## 1. 它是什么？

元数据映射是让不同引擎理解同一份数据表、字段、分区和权限的机制。

## 2. 它解决什么问题？

它解决多引擎访问时表定义不一致、字段含义不一致和治理割裂的问题。

## 3. 它在整个流程中的位置？

它连接 Paimon、Hive Metastore、Spark、Flink 和查询引擎。

## 4. 底层原理是什么？

通过 catalog、metastore 或同步机制维护库表、字段、分区、快照和权限信息。

## 5. 典型使用场景

Spark 读 Paimon 表、Flink 写湖表、查询引擎访问 ADS 数据。

## 6. 常见问题

字段变更不同步、表权限不一致、分区元数据延迟。

## 7. 优化方案

统一 catalog，规范 Schema 变更流程，建设元数据同步监控。

## 8. 和其他技术的区别

元数据映射关注“如何理解数据”，存储格式关注“数据如何保存”。
