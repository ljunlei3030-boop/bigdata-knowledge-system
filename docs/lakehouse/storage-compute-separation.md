# 存算分离

## 1. 它是什么？

存算分离是把数据存储和计算资源解耦的架构方式。

## 2. 它解决什么问题？

它解决计算扩缩容与存储容量绑定的问题，让不同引擎按需读取同一份数据。

## 3. 它在整个流程中的位置？

它是湖仓一体的基础能力，通常由 OSS / HDFS 加表格式共同支撑。

## 4. 底层原理是什么？

数据存储在统一存储层，Spark、Flink、ClickHouse / ByteHouse 等计算资源按任务读取和写入。

## 5. 典型使用场景

多引擎共享数据、弹性计算、冷热数据分层、成本优化。

## 6. 常见问题

对象存储延迟、元数据一致性、小文件和网络 IO 成本。

## 7. 优化方案

控制文件大小、缓存热点数据、优化分区和元数据访问。

## 8. 和其他技术的区别

存算分离是架构模式，不是单个存储或计算引擎。

## 9. 关联知识

- [Paimon](/storage/paimon)

## 总结输出

存算分离让数据平台更弹性，但需要用表格式、文件治理和缓存策略控制性能成本。
