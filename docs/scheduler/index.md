# 调度与运维概览

## 1. 它是什么？

调度与运维负责组织数据任务依赖、周期运行、补数、告警和故障恢复。

## 2. 它解决什么问题？

它解决数据链路如何稳定、按时、可观测运行的问题。

## 3. 它在整个流程中的位置？

它横向贯穿采集、入湖、加工、质量校验和数据服务。

## 4. 底层原理是什么？

通过 DAG 编排任务依赖，按照调度周期触发任务，并记录运行状态、日志和告警。

## 5. 典型使用场景

离线任务调度、指标产出、质量校验、失败重跑、历史补数。

## 6. 常见问题

依赖配置错误、补数影响下游、任务失败告警不及时。

## 7. 优化方案

规范依赖、设置 SLA、建立失败重试和质量校验节点。

## 8. 和其他技术的区别

调度关注任务运行组织，计算引擎关注任务如何执行。
