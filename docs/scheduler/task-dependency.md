# 任务依赖

## 1. 它是什么？

任务依赖描述一个数据任务在运行前需要等待哪些上游任务完成。

## 2. 它解决什么问题？

它解决数据产出顺序和数据可用性问题。

## 3. 它在整个流程中的位置？

它贯穿 ODS、DWD、DWS、ADS 以及质量校验任务。

## 4. 底层原理是什么？

调度系统通过 DAG 表达依赖关系，并按完成状态触发下游。

## 5. 典型使用场景

先入湖再清洗，先 DWD 再 DWS，先质量校验再发布服务数据。

## 6. 常见问题

依赖过宽导致等待过久，依赖过窄导致读取脏数据。

## 7. 优化方案

按数据真实依赖配置，区分强依赖、弱依赖和质量依赖。

## 8. 和其他技术的区别

任务依赖是运行组织能力，不是计算逻辑。

## 9. 关联知识

## 总结输出

任务依赖的目标是让数据按正确顺序稳定产出，同时避免不必要等待。
