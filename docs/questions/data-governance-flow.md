# 问题：数据治理在整个数据流程中做了什么？

## 答案

数据治理在采集、入湖、建模、加工、服务、使用和监控环节保障数据可理解、可信、可追踪、可授权和可控成本。

## 背后原理

治理通过元数据、血缘、质量、权限和成本机制把数据链路连接起来。

## 发生场景

新增数据源、指标异常、权限申请、任务成本过高、质量波动。

## 排查方式

查看数据资产目录、血缘链路、质量规则、权限记录和任务成本。

## 解决方案

建立采集规范、表负责人、质量规则、血缘追踪、权限分级和成本看板。

## 方案代价

治理需要流程和工具维护，短期会增加规范成本，但长期降低平台混乱度。

## 总结输出

数据治理的作用是让数据平台长期可信，而不是只在数据出错后补救。
