# 问题：ClickHouse 查询单表为什么快？

## 答案

ClickHouse 单表查询快，主要来自列式存储、压缩、向量化执行、主键排序和数据跳过能力。

## 背后原理

查询只读取需要的列，并通过分区、排序键和索引跳过无关数据块。

## 发生场景

大宽表明细查询、固定维度筛选、报表指标查询。

## 排查方式

查看查询扫描行数、分区命中、排序键是否生效和读取列数量。

## 解决方案

让分区键、排序键和查询条件匹配，减少无效列和无效扫描。

## 方案代价

排序键设计不合理会影响写入和查询，预聚合会增加存储和维护成本。

## 总结输出

ClickHouse 快的前提是数据布局匹配查询模式，而不是任何 SQL 都天然快。
