# 数据存储概览

## 1. 它是什么？

数据存储负责承载数据平台中的原始数据、明细数据、汇总数据和服务数据。

## 2. 它解决什么问题？

它解决数据如何长期保存、如何组织表结构、如何被多引擎读取和如何控制成本的问题。

## 3. 它在整个流程中的位置？

数据存储位于采集之后、计算之前，同时也是治理、查询和服务的基础。

## 4. 底层原理是什么？

常见组合包括 HDFS / OSS 承载文件，Parquet / ORC 存储列式数据，Hive 或 Paimon 管理表和元数据。

## 5. 典型使用场景

ODS 明细存储、DWD 宽表、Paimon 增量湖表、ADS 查询结果。

## 6. 常见问题

小文件过多、分区设计不合理、元数据膨胀、存储成本上升。

## 7. 优化方案

规划分区、文件大小、生命周期、压缩格式和表格式。

## 8. 和其他技术的区别

存储关注数据组织和持久化，计算关注执行和加工。

## 9. 关联知识

- [Paimon](/storage/paimon)
- [Parquet / ORC](/storage/parquet-orc)

## 总结输出

存储设计决定后续计算效率和治理成本，核心是文件、表、分区、元数据和生命周期。
