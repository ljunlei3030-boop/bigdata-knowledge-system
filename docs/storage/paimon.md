# Paimon

## 1. 它是什么？

Paimon 是湖仓表格式，用于在数据湖中管理文件、快照、增量变更和元数据。

## 2. 它解决什么问题？

它解决传统湖存储难以支持更新、增量消费和流批一体的问题。

## 3. 它在整个流程中的位置？

Paimon 位于湖仓一体的存储层，上游接收 Flink CDC 或 Kafka 入湖数据，下游被 Spark、Flink 和查询引擎消费。

## 4. 底层原理是什么？

Paimon 通过表格式组织数据文件、变更记录和快照，让数据可以被批读、流读和增量读。

## 5. 典型使用场景

数据库实时入湖、ODS 增量表、DWD 明细更新、湖表多引擎访问。

## 6. 常见问题

主键表设计、文件合并、快照管理、写入延迟和查询性能。

## 7. 优化方案

合理设计主键、分区、bucket、compaction 和快照保留策略。

## 8. 和其他技术的区别

Hive 偏离线表管理，Paimon 更适合变更数据和流批一体场景。
