# HDFS / OSS

## 1. 它是什么？

HDFS 和 OSS 都是大数据场景中的底层存储介质，分别代表分布式文件系统和对象存储。

## 2. 它解决什么问题？

它们解决海量数据低成本、可扩展、长期保存的问题。

## 3. 它在整个流程中的位置？

它们承载 Hive、Paimon、Parquet、ORC 等数据文件。

## 4. 底层原理是什么？

HDFS 以块和副本管理数据，OSS 以对象方式管理数据，计算引擎通过文件或对象接口读取。

## 5. 典型使用场景

离线数仓、数据湖、历史归档、低成本存储。

## 6. 常见问题

小文件、目录过多、元数据压力、对象存储读写延迟。

## 7. 优化方案

合并小文件、控制分区目录、使用列式格式和合理生命周期策略。

## 8. 和其他技术的区别

HDFS / OSS 是存储底座，Paimon / Hive 在其上组织表和元数据。

## 总结输出

HDFS / OSS 决定数据平台的容量和成本边界，表格式和文件格式决定数据如何被高效使用。
