# Parquet / ORC

## 1. 它是什么？

Parquet 和 ORC 都是大数据场景常用的列式存储格式。

## 2. 它解决什么问题？

它们通过列式组织、压缩和统计信息提升扫描效率并降低存储成本。

## 3. 它在整个流程中的位置？

它们通常作为 Hive、Paimon 或离线表的底层文件格式。

## 4. 底层原理是什么？

列式格式按列存储数据，查询只读取需要的列，并利用统计信息跳过不必要的数据块。

## 5. 典型使用场景

宽表查询、指标汇总、离线分析、湖仓数据存储。

## 6. 常见问题

小文件过多、压缩算法选择不当、字段过宽导致扫描成本高。

## 7. 优化方案

控制文件大小、裁剪字段、选择合适压缩算法、结合分区和排序。

## 8. 和其他技术的区别

Parquet / ORC 是文件格式，Hive / Paimon 是表管理能力。

## 总结输出

列式文件格式是大数据查询性能的基础，关键价值是少读列、压缩好、可跳过。
